# アイスブレイクツール 受け入れ基準

## 概要

このドキュメントはアイスブレイクツールの受け入れ基準とテスト項目を記載します。

## 機能テスト基準

**【信頼性レベル凡例】**:
- 🔵 **青信号**: ユーザヒアリングを参考にした確実なテスト基準
- 🟡 **黄信号**: ユーザヒアリングから妥当な推測によるテスト基準
- 🔴 **赤信号**: ユーザヒアリングにない推測によるテスト基準

---

### REQ-001: ランダム質問表示 の受け入れ基準 🔵

**Given（前提条件）**:
- ユーザーがブラウザでツールを開いている

**When（実行条件）**:
- ページが表示された直後

**Then（期待結果）**:
- 質問プールからランダムに1つの質問が表示される
- 質問文が画面中央に大きく表示される

**テストケース**:
- [ ] 正常系: ページを開くと質問が1問表示される
- [ ] 正常系: ページをリロードすると（同じか異なる）質問が表示される
- [ ] 正常系: 表示される質問はユーモア・軽い系のもの

---

### REQ-101: 「次の質問」ボタン の受け入れ基準 🔵

**Given（前提条件）**:
- 質問が1問表示されている状態

**When（実行条件）**:
- ユーザーが「次の質問」ボタンを押す

**Then（期待結果）**:
- 新しいランダムな質問が即座に表示される（100ms以内）
- 前の質問は履歴に追加される

**テストケース**:
- [ ] 正常系: ボタンを押すと新しい質問に切り替わる
- [ ] 正常系: 繰り返し押しても毎回質問が切り替わる
- [ ] 境界値: 同じ質問が連続して出ることを許容する（重複排除不要）

---

### REQ-003/REQ-102: 履歴表示 の受け入れ基準 🔵

**Given（前提条件）**:
- 1問以上の質問が表示された状態

**When（実行条件）**:
- 「次の質問」ボタンが押された後

**Then（期待結果）**:
- 前に表示された質問が履歴リストに追加される
- 履歴は表示された順（新しい順 or 古い順）で一覧表示される

**テストケース**:
- [ ] 正常系: 最初の質問ボタン押下後、履歴に1件追加される
- [ ] 正常系: 複数回押すと押した回数分の履歴が表示される
- [ ] 正常系: ページリロード後は履歴がリセットされる（セッション内のみ）
- [ ] 境界値: 同じ質問が複数回履歴に登録されることを許容する

---

### REQ-004: ポップ・カラフルUI の受け入れ基準 🔵

**Given（前提条件）**:
- ツールをブラウザで表示している状態

**When（実行条件）**:
- 任意のタイミング（表示直後・操作中）

**Then（期待結果）**:
- 背景・ボタン・文字などにカラフルな配色が使われている
- 画面共有時に参加者が「楽しそう」と感じるデザインである

**テストケース**:
- [ ] 正常系: ページに複数の明るい色が使われている
- [ ] 正常系: ボタンが目立つデザインになっている
- [ ] 正常系: 質問文が読みやすいフォントサイズで表示されている

---

## 非機能テスト基準

### パフォーマンステスト

**NFR-001/002: 表示速度 🟡**

- [ ] 初回ページ表示: 3秒以内に完了
- [ ] 「次の質問」ボタン応答: 即時（100ms以内）

**テスト方法**:
- Chrome DevTools の Lighthouse でパフォーマンス計測
- 合格基準: Lighthouse パフォーマンススコア 70以上

### ユーザビリティテスト

**NFR-201: レスポンシブデザイン 🟡**

- [ ] スマートフォン（375px幅）での正常表示
- [ ] タブレット（768px幅）での正常表示
- [ ] PC（1280px幅）での正常表示
- [ ] 質問文・ボタンが各デバイスで操作しやすいサイズである

**NFR-202: 直感的操作性 🔵**

- [ ] 説明なしで「次の質問」ボタンが分かる
- [ ] ボタンは画面の目立つ位置に配置されている

---

## Edgeケーステスト基準

### EDGE-101: 重複質問の許容 🔵

**テストシナリオ**:
- 多数回「次の質問」を押し続けた場合も正常動作すること

**合格基準**:
- [ ] システムがクラッシュしない
- [ ] 同じ質問が表示されても履歴に追加される
- [ ] エラーメッセージは表示されない（正常動作として扱う）
