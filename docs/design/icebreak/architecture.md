# アイスブレイクツール アーキテクチャ設計

## システム概要

オンライン会議向けアイスブレイクツール。
ランダムなユーモア・軽い系の日本語質問を1問ずつ表示し、履歴を管理するクライアントサイド完結型Webアプリ。

**【信頼性レベル凡例】**:
- 🔵 **青信号**: 要件定義書・技術スタックを参考にしてほぼ推測していない場合
- 🟡 **黄信号**: 要件定義書・技術スタックから妥当な推測の場合
- 🔴 **赤信号**: 要件定義書・技術スタックにない推測の場合

---

## アーキテクチャパターン

- **パターン**: シングルページアプリケーション（SPA） + 静的サイト生成（SSG） 🔵 *REQ-402（外部API不使用）・Vercel無料デプロイ要件より*
- **理由**: 外部APIとDBを使用しないため、クライアントサイドのみで完結する最もシンプルな構成を採用。Vercel無料プランでの永続デプロイに最適。

---

## コンポーネント構成

### フロントエンド 🔵 *要件定義書 技術スタックより*

| 項目 | 採用技術 | 理由 |
|------|----------|------|
| フレームワーク | Next.js 14（App Router） | ユーザー指定 |
| 言語 | TypeScript | ユーザー指定 |
| スタイリング | Tailwind CSS | ユーザー指定 |
| 状態管理 | React useState（ローカル状態） | 外部状態管理不要のシンプルな構成 |
| データ | 静的 TypeScript 配列 | REQ-402（外部API不使用） |

### バックエンド

- **不要** 🔵 *REQ-402（外部APIとデータベース不使用）より*
- 全ロジックはクライアントサイド（ブラウザ上）で完結する

### データベース

- **不要** 🔵 *REQ-402（外部APIとデータベース不使用）より*
- 質問データは `src/data/questions.ts` に静的配列として保持
- 履歴は React の `useState` でセッション中のみ保持（ページリロードでリセット）

## 状態管理設計

```
useIceBreak（カスタムフック）
├── currentQuestion: Question     # 現在表示中の質問
├── history: Question[]           # 表示済み質問の履歴
├── questionNumber: number        # 現在何問目か（REQ-302）
└── nextQuestion(): void          # 次のランダム質問を取得（REQ-101・REQ-102）
```

🟡 *React カスタムフックによる状態集約は Next.js App Router のベストプラクティスから推測*

## デプロイ構成

```
GitHub リポジトリ
      ↓ push / PR merge
Vercel（自動デプロイ）
      ↓
https://{project-name}.vercel.app
```

🔵 *REQ-401（Vercel無料プラン）より*

- GitHub と Vercel を連携し、`main` ブランチへのプッシュで自動デプロイ
- 静的サイトとして生成されるため、Vercel 無料プランの制約に完全適合
- カスタムドメイン不要であれば、`*.vercel.app` で永続的に公開可能

---

## 技術的制約事項

| 制約 | 内容 | 対応 |
|------|------|------|
| Vercel無料プラン | 商用利用制限・帯域制限 | 個人利用範囲で問題なし |
| データ永続化なし | セッションリセットで履歴消失 | REQ-103で許容済み |
| 外部API不使用 | 質問の動的追加不可 | 静的配列で管理（REQ-402） |
